# KWP2000 — developer notes

Bench results, connect semantics, CH340 slow-init behavior, and settings persistence. For end-user limits see [README.md](../README.md).

**Scope:** Direct K-line to ECU on bench (no cluster). Bootmode is separate ([README](../README.md), issue #44).

## Results (defaults)

Slow init address **`0x01`**; fast init physical **`0x01`**.

| ECU | Adapter | Slow init | Fast init |
| --- | --- | --- | --- |
| ME7.1 | FTDI | ✅ | ✅ |
| ME7.1 | CH340 | ✅ | ✅ |
| ME7.5 | FTDI | ✅ | ❌ |
| ME7.5 | CH340 | ✅ | ❌ |

**Bench guidance:** use **slow init** @ **`0x01`**. ME7.5 fast init fails on this unit (not an adapter issue — slow init works on the same K-line). Other flash images may differ; no auto fallback in code.

## Connect UI

One **Connect address** field; meaning depends on **Slow init / Fast init** on the connect tab:

| Connect method | Address | Address mode |
| --- | --- | --- |
| Slow init | 5-baud byte (`0x01` on ME7 bench) | Ignored |
| Fast init | StartCommunication target | Physical / Functional |

**Internal retry:** after KWP1281 @ **`0x01`**, slow init may retry @ **`0x11`** (Even parity). Do not use **`0x11`** as a user one-shot address — no sync on bench.

Settings tab hides **Fast init address mode** when connect tab uses slow init.

---

## CH340 slow init

Slow init uses **per-bit `SetBreak` toggles** (~200 ms/bit, 7O1 Odd) in `SendSlowInitAddress_UsingBreak`. Fast init uses a **single** short break — different path.

Production flow: `Connect_SlowInit()` → CH340 KKL DTR pulse + line restore → break-toggle send → optional UART fallback if break send fails → `WaitForTransmitDrain` before line changes.

Use **Slow init timing log** in settings when comparing CH340 vs FTDI bit timing. Sync wait **`W1Max` 450 ms** (ECU `0x55` ~301 ms after address on ME7.1 FTDI bench).

## Settings persistence

Saved to `%AppData%` on exit. Defaults: `KWP2000SettingsDefaults.cs`.

**Remembered:** connect-tab init method, baud, detected baud list, slow-init timing log.

**Session-only:** connect address, fast-init address mode, timing offsets, device timeouts, P1–P4, security options, **`NumConnectionAttempts`**, etc.

**Restore To Defaults** resets the settings tab only — not connect-tab init method or baud.

Legacy **`AllowCh340SlowInit`** is removed (ignored in old user.config). CH340 slow init is always on.

## Bench checklist

- Adapter in dumb mode; K-line direct to ECU; stable power
- Slow init @ **`0x01`**
- Wait **≥ 2.6 s** between connect attempts
- Failures: `LoggerLog.txt` (AppData)
